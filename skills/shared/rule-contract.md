# Rule contract

- Canon rules use IDs `rule.<domain>.<slug>` in `content/canon/`.
- References use `ref.<owner>.<slug>` in `content/references/`.
- Artifacts claim implemented rules in `content/artifacts/`.
- Agents must cite rule IDs when producing UI guidance and stay inside `@awesome-ds/react` + tokens.
- Social findings stay in `content/signals/` until human promotion.
